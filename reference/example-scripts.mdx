JSBSim simulation scripts are XML files that define a complete, repeatable simulation run: which aircraft to load, initial conditions, simulation duration, and a sequence of timed events that manipulate controls and log data. They let you run the same flight scenario identically every time without writing any code.

## Running a script

<Tabs>
  <Tab title="Command line">
    ```bash
    JSBSim --script=scripts/c1721.xml
    ```

    You can override properties at the command line:

    ```bash
    JSBSim --script=scripts/c1723.xml --property=simulation/randomseed=42
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import jsbsim

    fdm = jsbsim.FGFDMExec(None)
    fdm.load_script('scripts/c1721.xml')
    fdm.run_ic()

    while fdm.run():
        pass
    ```
  </Tab>
</Tabs>

## Script XML structure

A JSBSim script file uses the `<runscript>` root element. Here is a complete working script (`c1721.xml`) annotated to explain each element:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<runscript xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:noNamespaceSchemaLocation="http://jsbsim.sf.net/JSBSimScript.xsd"
    name="C172-01A">

  <!-- Which aircraft model and initial conditions file to load -->
  <use aircraft="c172r" initialize="reset00"/>

  <!-- Simulation time span and integration timestep (seconds) -->
  <run start="0.0" end="10" dt="0.008333">

    <!-- Events fire when their <condition> becomes true -->
    <event name="event name">
      <condition>simulation/sim-time-sec  ge  0.25</condition>
      <!-- FG_STEP: step to value over tc seconds -->
      <set name="fcs/aileron-cmd-norm" action="FG_STEP" value="0.25" tc="0.25"/>
    </event>

    <event name="event name">
      <condition>simulation/sim-time-sec  ge  0.5</condition>
      <!-- FG_EXP + FG_DELTA: exponential ramp, relative to current value -->
      <set name="fcs/aileron-cmd-norm" action="FG_EXP" type="FG_DELTA" value="0.5" tc="0.5"/>
    </event>

    <event name="event name">
      <condition>simulation/sim-time-sec  ge  1.5</condition>
      <!-- FG_RAMP: linear ramp to value over tc seconds -->
      <set name="fcs/rudder-cmd-norm" action="FG_RAMP" type="FG_DELTA" value="0.5" tc="0.5"/>
    </event>

    <event name="event name">
      <condition>simulation/sim-time-sec  ge  1.5</condition>
      <set name="fcs/elevator-cmd-norm" action="FG_RAMP" value="0.25" tc="0.5"/>
    </event>

    <event name="event name">
      <condition>simulation/sim-time-sec  ge  2.5</condition>
      <set name="fcs/elevator-cmd-norm" action="FG_EXP" type="FG_DELTA" value="0.5" tc="0.5"/>
    </event>

  </run>
</runscript>
```

### Script elements reference

| Element | Purpose |
|---------|---------|
| `<use aircraft="..." initialize="...">` | Selects the aircraft model folder and the `reset` initialization file inside it. |
| `<run start end dt>` | Sets the simulation window. `dt` is the integration timestep in seconds (120 Hz = 0.008333 s). |
| `<property value="...">` | Sets a property at simulation start, before events begin. |
| `<event name="..." persistent="true">` | Defines a triggered action. `persistent="true"` means the event re-arms after it fires. |
| `<condition>` | A boolean expression evaluated each timestep. Uses property names and operators `ge`, `le`, `gt`, `lt`, `eq`, `ne`. |
| `<delay>` | Number of seconds to wait after the condition becomes true before firing the event. |
| `<set name action value tc type>` | Sets a property. `action` controls the shape: `FG_STEP`, `FG_RAMP`, or `FG_EXP`. `type="FG_DELTA"` applies the value relative to the current property value. `tc` is the time constant in seconds. |
| `<notify>` | Prints a snapshot of listed properties to stdout when the event fires. |

## A complete example: c1723.xml (takeoff and autopilot climb)

`c1723.xml` is the most complete introductory script in the bundle. It loads the c172x, starts the engine, releases the brakes, rotates at 51 KEAS, and then hands control to the autopilot to climb and capture a heading. Key patterns it demonstrates:

- Using XML entities (`<!ENTITY on "1">`) as named constants for readability
- Initializing properties before events using bare `<property>` elements
- Condition-triggered events (time-based and airspeed-based)
- `<delay>` to add a pause between condition becoming true and the event firing
- `persistent="true"` events that repeat on a timer
- `<notify>` blocks that print structured telemetry to stdout

<Tip>
  Open `c1723.xml` alongside the [property system reference](/concepts/property-system) to understand how each `<set>` target maps to a simulation variable.
</Tip>
