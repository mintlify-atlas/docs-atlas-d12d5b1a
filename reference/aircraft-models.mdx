JSBSim ships with 61 aircraft models in the `aircraft/` directory. Each model is a self-contained folder of XML files describing the airframe, aerodynamics, propulsion, and flight control system. You can use these models as-is for simulation and testing, or as a starting point for your own aircraft.

## Loading an aircraft

Load any bundled aircraft by name using `load_model`:

```python
import jsbsim

fdm = jsbsim.FGFDMExec(None)
fdm.load_model('c172p')
fdm.run_ic()
```

You can also load an aircraft from the command-line executable:

```bash
JSBSim --aircraft=c172p --initfile=reset00
```

<Note>
  Pass the root path to `FGFDMExec` if you are working outside the installed package directory, e.g. `jsbsim.FGFDMExec('/path/to/jsbsim-root')`.
</Note>

## Creating your own aircraft

Use `aircraft_template.xml` in the `aircraft/` directory as the starting point for a new aircraft definition. It provides the required XML skeleton with all top-level elements in place.

```bash
cp aircraft/aircraft_template.xml aircraft/myplane/myplane.xml
```

The template includes placeholder sections for:

- `<metrics>` — reference area, span, chord, and aerodynamic reference point
- `<mass_balance>` — empty weight, CG location, and moments of inertia
- `<aerodynamics>` — lift, drag, side force, roll, pitch, and yaw coefficients
- `<propulsion>` — engine and thruster definitions
- `<flight_control>` — FCS channels and components
- `<ground_reactions>` — contact point definitions

<Tip>
  Start from a similar existing aircraft rather than the blank template. For example, copy `c172p` as the base for another single-engine piston aircraft to inherit a complete, working aerodynamic dataset you can then modify.
</Tip>
