## What is the property system?

JSBSim uses a hierarchical key-value store called the property tree to share simulation state between models. Every quantity that a model produces or consumes — altitude, airspeed, control surface position, engine thrust — is bound to a named property node. Models communicate by reading and writing these nodes rather than calling each other directly.

The tree is managed by `FGPropertyManager`, which wraps SimGear's `SGPropertyNode`. Property paths are Unix-style slash-separated strings:

```
position/h-sl-ft
velocities/vt-fps
attitude/phi-rad
fcs/aileron-cmd-norm
propulsion/engine[0]/thrust-lbs
```

Indexed items — such as engines or gear legs — use bracket notation: `propulsion/engine[0]/thrust-lbs`, `contact/unit[1]/WOW`.

## Reading and writing properties

### C++

`FGFDMExec` provides `GetPropertyValue()` and `SetPropertyValue()` for convenient scalar access:

```cpp
// Read altitude above sea level (ft)
double altitude = fdmex->GetPropertyValue("position/h-sl-ft");

// Write a throttle command
fdmex->SetPropertyValue("fcs/throttle-cmd-norm", 0.75);
```

Internally these call `getDoubleValue()` / `setDoubleValue()` on the root `SGPropertyNode`.

### Python

The Python binding exposes the same operations via dictionary-style subscript syntax on `FGFDMExec`:

```python
import jsbsim

fdm = jsbsim.FGFDMExec("path/to/jsbsim/root")
fdm.load_model("c172p")
fdm.run_ic()

while fdm.run():
    # Read properties
    altitude_ft  = fdm["position/h-sl-ft"]
    airspeed_fps = fdm["velocities/vt-fps"]
    bank_rad     = fdm["attitude/phi-rad"]

    # Write properties
    fdm["fcs/throttle-cmd-norm"] = 0.8
    fdm["fcs/aileron-cmd-norm"]  = 0.1
```

You can also use the explicit methods `get_property_value()` and `set_property_value()`:

```python
altitude = fdm.get_property_value("position/h-sl-ft")
fdm.set_property_value("fcs/elevator-cmd-norm", -0.05)
```

## FGPropertyNode and FGPropertyManager

For lower-level access, the Python API exposes `FGPropertyNode` and `FGPropertyManager` directly.

```python
import jsbsim

# Get the property manager from a running executive
pm = fdm.get_property_manager()

# Navigate to a specific node by path
node = pm.get_node("position/h-sl-ft")

if node:
    print(node)                     # "Property 'position/h-sl-ft' (value: 8000.0)"
    value = node.get_double_value() # 8000.0
    node.set_double_value(9000.0)   # write a new value

# Check if a property exists before accessing it
if pm.hasNode("propulsion/engine[1]/thrust-lbs"):
    thrust = pm.get_node("propulsion/engine[1]/thrust-lbs").get_double_value()
```

`FGPropertyNode` can also traverse the tree relatively:

```python
# Get a child node from a parent node
fcs_node  = pm.get_node("fcs")
aileron   = fcs_node.get_node("aileron-cmd-norm")

name = aileron.get_name()                   # "aileron-cmd-norm"
fqn  = aileron.get_fully_qualified_name()   # "fcs/aileron-cmd-norm"
```

<Warning>
  `get_fully_qualified_name()` is described in the source as "very slow" and is intended for debugging only. Avoid calling it in tight simulation loops.
</Warning>

## Read/write attributes

Each property node has access mode flags defined by the `Attribute` enum:

```python
import jsbsim

class Attribute(enum.Enum):
    NO_ATTR = 0
    READ    = 1
    WRITE   = 2
```

You can check or set these flags on any `FGPropertyNode`:

```python
node = pm.get_node("fcs/aileron-cmd-norm")

is_readable = node.get_attribute(jsbsim.Attribute.READ)   # True
is_writable = node.get_attribute(jsbsim.Attribute.WRITE)  # True

# Make a node read-only
node.set_attribute(jsbsim.Attribute.WRITE, False)
```

Some properties are write-only by design (for example, `simulator/do_trim` accepts a trim mode integer but produces no readable output).

## Discovering properties at runtime

`FGFDMExec` can enumerate all bound properties for a loaded aircraft:

```cpp
// C++: query the catalog for all properties matching a prefix
std::string results = fdmex->QueryPropertyCatalog("fcs/");

// Print the entire property catalog
fdmex->PrintPropertyCatalog();
```

```python
# Python: query properties matching a string
catalog = fdm.query_property_catalog("propulsion/")
print(catalog)
```

This is useful for discovering what properties a particular aircraft model exposes before writing control scripts or output directives.

<Tip>
  Property paths are case-sensitive. Use the exact path as reported by `PrintPropertyCatalog()` or from the source `@property` annotations in the model headers.
</Tip>
